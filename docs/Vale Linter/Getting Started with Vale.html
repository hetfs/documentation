Vale Lint for Documentation

Vale is an open-source linting tool that enhances the clarity, consistency, and quality of written content. It enables teams to enforce custom style guides and best writing practices, making it indispensable for technical writers, editors, and content creators. Whether you're managing software documentation, blog posts, or knowledge bases, Vale ensures your content aligns with predefined standards.

Key Features​

🔹 Customizable Style Guides​

  • Define custom rules tailored to your organization’s writing standards.
  • Use built-in style guides like Proselint, Write-good, and Alex, or create your own.
  • Enforce consistent terminology, refine brand voice, and eliminate jargon and biased language.

🔹 Real-Time Feedback​

  • Provides instant linting to catch errors as you write.
  • Detects grammatical mistakes, unclear phrasing, and inconsistencies before publication.
  • Reduces manual editing, streamlining the content review process.

🔹 Seamless Integration with Editors & Workflows​

  • Works with popular text editors and IDEs, including:
    ✅ Visual Studio Code (VS Code)
    ✅ Sublime Text
    ✅ Atom
    ✅ Neovim
  • Supports CI/CD automation via GitHub Actions, GitLab CI/CD, and pre-commit hooks, ensuring writing consistency across development pipelines.

🔹 Multi-Format Compatibility​

  • Supports multiple file formats:
    ✅ Markdown (.md) – For documentation and blogs.
    ✅ reStructuredText (.rst) – Common for Python documentation.
    ✅ HTML (.html) – For static website content.
    ✅ LaTeX (.tex) – Useful for academic and technical papers.
  • Ensures uniform writing quality across different content types.

🔹 Lightweight & Automation-Friendly​

  • Fast and efficient, ideal for CI/CD pipelines.
  • Works seamlessly with GitHub Actions, GitLab CI/CD, Travis CI, and other automation tools.
  • Supports batch processing for large-scale content reviews.

Use Cases​

✅ Technical Documentation​

  • Standardizes terminology and formatting in API docs, guides, and manuals.
  • Identifies and corrects ambiguities, typos, and inconsistencies.

✅ Content Editing & Review​

  • Automates content reviews to align with brand guidelines.
  • Helps writers and editors avoid common mistakes and improve clarity.

✅ Documentation Quality Assurance​

  • Standardizes documentation for open-source projects, enterprise wikis, and internal knowledge bases.
  • Reduces manual review cycles by automating style enforcement.

Why Choose Vale?​

✅ Open-Source & Customizable – Free and flexible, supporting custom rules.
✅ Lightweight & Fast – Runs with minimal dependencies.
✅ Versatile & Well-Supported – Compatible with multiple editors, file formats, and CI/CD workflows.
✅ Trusted by Industry Experts – Used by leading companies, open-source projects, and documentation teams.

Getting Started with Vale​

1️⃣ Install Vale​

brew install vale  # macOS (Homebrew)
winget install vale # Windows (winget or Chocolatey)
pip install vale   # Python users (via pip)

After installation, create a .vale.ini file in your project’s root directory to configure Vale’s behavior.

For alternative installation methods, visit the official Vale documentation.

Vale Config Generator​

The Vale Configuration Generator simplifies the creation of .vale.ini, defining writing rules, alert levels, and applicable file formats.

📌 Use the Config Generator to generate a .vale.ini file.

1. Configuration Overview​

This tool helps you:
✔ Select a base style (primary writing guide).
✔ Add supplementary styles (e.g., inclusivity, readability).
✔ Configure file formats (Markdown, MDX, reStructuredText).
✔ Generate a .vale.ini file to enforce writing standards.

2. Choosing a Base Style​

Common base styles include:

  • Microsoft Writing Style Guide
  • Google Developer Documentation Style Guide
  • Red Hat Documentation Style Guide

Use only one base style to prevent conflicts.

Example: Using Microsoft Writing Style Guide

StylesPath = styles
MinAlertLevel = suggestion
Packages = Microsoft

[*.{md}]
BasedOnStyles = Vale, Microsoft

✔ Effect: The Microsoft Writing Style Guide applies to all .md files.

💡 Tip: Run vale sync after creating .vale.ini to initialize Vale.

3. Adding Supplementary Styles​

Supplementary styles enhance content checks:

  • Proselint – Grammar & readability improvements.
  • Write-good – Readability enforcement.
  • Alex – Detects non-inclusive language.
  • Joblint – Ensures bias-free job descriptions.

Example: Combining multiple styles

StylesPath = styles
MinAlertLevel = suggestion
Packages = Microsoft, proselint, write-good, alex

[*.{md}]
BasedOnStyles = Vale, Microsoft, proselint, write-good, alex

✔ Effect:

  • Microsoft – Writing best practices.
  • Proselint – Grammar & readability improvements.
  • Write-good – Clearer, concise writing.
  • Alex – Bias-free language enforcement.

4. Format-Specific Configurations​

Tailor Vale’s behavior for specific file formats or frameworks.

Examples​

Markdown & MDX Configuration

StylesPath = styles
MinAlertLevel = suggestion
Packages = Microsoft, proselint, write-good, alex, Hugo, MDX

[*.{md,mdx}]
BasedOnStyles = Vale, Microsoft, proselint, write-good, alex

✔ Effect: Applies style rules to both Markdown (.md) and MDX (.mdx) files.

Markdown & reStructuredText (.rst) Configuration

[*.{md,rst}]
BasedOnStyles = proselint

✔ Effect:

  • Proselint rules apply to Markdown (.md) and reStructuredText (.rst).
  • .rst is widely used in Python documentation.

5. Running Vale​

After configuring .vale.ini, initialize Vale to download styles and apply rules.

🔹 Sync Vale with Configuration

vale sync

✔ Downloads required style guides from StylesPath.

🔹 Explore More Styles
Check the Package Explorer for additional styles and configurations.
