Markdown-and-tools

Markdown Basics​

Markdown is a simple language for formatting plain text, widely used on platforms like GitHub to style README files, pull requests, and issue comments. It offers features like bold, italic, lists, links, and images. Its goal is to make text easy to read and write without complex syntax.

Markdown was developed by John Gruber and Aaron Swartz in 2004. Gruber created the first markdown-to-HTML converter in Perl, and by 2014, there were many implementations in different languages.

John Gruber's description of Markdown is intentionally vague, leading to various implementations. Tools like MDTest and Babelmark3 help resolve inconsistencies by comparing Markdown outputs from different implementations.

CommonMark Adoption​

The CommonMark specification has been adopted by many platforms:

  • GitHub

MarkdownLint Tool​

Markdownlint checks Markdown files for style issues, ensuring consistent formatting. It detects problems like incorrect indentation or header structure.

Installation​

Markdownlint is available via package managers and RubyGems:

gem install mdl

Usage​

To lint files, run:

mdl README.md

To lint a directory:

mdl docs/

For more options, use:

mdl --help

Markdownlint lists issues by line number. Check RULES.md for more details.

Markdownlint has many more options you can pass on the command line, run mdl --help to see what they are, or see the documentation on configuring markdownlint.

Styles​

Markdown comes in various flavors and styles, all of which are valid. Markdownlint's default settings reflect the author's preferences, but your project may have its own guidelines.

Markdownlint doesn't enforce a single style but supports style files, which allow you to enable specific rules and customize settings. For example, rule MD013 flags lines longer than 80 characters, but this limit can be adjusted or disabled in a style file.

If the default rules don't meet your needs, you can create custom rules and rulesets.

For details, see the creating styles and creating rules documents.

CommonMark Implementations

Recently Maintained Projects​

C​

  • cmark: Reference implementation with a high-performance executable and library.
  • cmark-gfm: GitHub's fork of cmark, includes extensions for tables, task lists, and more.
  • MD4C: Fast parser implementation with a minimalistic interface, akin to a SAX parser in XML parsing.

C#​

  • CommonMark.NET: Available on NuGet.
  • Markdig: Feature-rich and extensible, available on NuGet.

Common Lisp​

  • cl-cmark: Wrapper for the C library.

Crystal​

  • markd: Pure Crystal implementation.

D​

  • commonmark-d: Port of MD4C.
  • hunt-markdown: Port of the commonmark-java library.

Dart​

  • markdown: Available on Pub.

Elixir​

  • Cmark: Wraps the C library.

Emacs Lisp​

  • cmark: Port of commonmark.js.

Go​

  • golang-commonmark/markdown: Pure Go implementation.
  • go-commonmark: Wrapper for the C library.
  • goldmark: Pure Go implementation.
  • Lute: Pure Go implementation.

Haskell​

  • cmark: Wrapper for the C library.
  • commonmark-hs: Pure Haskell implementation.

Haxe​

  • necmark: Wrapper for the C library with a neko target.

Java​

  • commonmark-java: Pure Java implementation.
  • flexmark-java: Rich features and extensible down to single characters.

JavaScript​

  • commonmark.js: Reference implementation. Install via npm install commonmark.
  • markdown-it: High-speed and pluggable. Install via npm install markdown-it.
  • cmark-emscripten: Port of the C library to WebAssembly. Install via npm install cmark-emscripten.
  • mdx: Superset of CommonMark with embedded JSX and import/export syntax.
  • ciceromark: Extends the CommonMark DOM for smart legal contracts.

Julia​

  • CommonMark.jl: Pure Julia implementation with extensions.

Lua​

  • cmark-lua: Wraps the C library. Install via luarocks install cmark.
  • lcmark: Higher-level interface around cmark with YAML metadata, filters, and templates.

LuaTeX​

  • The Markdown Package for TeX: Rewritten lunamark Lua library for typesetting CommonMark in TeX.

OCaml​

  • Cmarkit: Pure OCaml implementation.

Perl​

  • CommonMark: Wraps the C library.
  • pmarkdown: Pure Perl implementation.

PHP​

  • league/commonmark: Highly extensible, with support for custom extensions.
  • cmark: PECL extension based on the original cmark implementation.

Python​

  • markdown-it-py: High-speed, pluggable port of the JS markdown-it library.
  • mistletoe: Fast, extensible, pure Python implementation.
  • Marko: Pure Python, focused on easy extensibility.

Ruby/RubyMotion​

  • CommonMarker: Wrapper for the C library.
  • motion-markdown-it: Port of JS markdown-it library for Ruby and RubyMotion.

R​

  • commonmark: Wrapper for the C library.

Racket​

  • commonmark: Native Racket implementation with full test coverage.

Rust​

  • comrak: Based on cmark-gfm with GitHub extensions.
  • pulldown-cmark: Performance-oriented architecture.
  • markdown-rs: CommonMark-compliant parser with ASTs and extensions.

Scala​

  • cmark-scala: Wrapper for the C library.

Scheme​

  • guile-commonmark: GNU Guile implementation.

Swift​

  • Down: Wraps the reference cmark library.
  • SwiftMark: Wraps the C library.
  • MarkdownSyntax: Access to AST via the reference C library.

Tcl​

  • tcl-cmark: Wrapper for cmark-gfm.

TypeScript​

  • DefinitelyTyped: Provides JS library bindings.
  • FlavorMark: Modularized rewrite of commonmark.js.

Zig​

  • koino: Based on Comrak/cmark-gfm, with GitHub extensions.
